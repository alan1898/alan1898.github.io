CPA IBE with RO

Posted on 2020-09-05

Some CPA-Secure Identity-Based Encryption Schemes with Random Oracles.

Boneh-Franklin Scheme

Construction

  • Setup$(1^\lambda)\rightarrow(pp,mk)$
  • KeyGen$(I\in\{0,1\}^\ast,mk)\rightarrow d_I$
  • Encrypt$(m\in\{0,1\}^n,I)\rightarrow CT$
  • Decrypt$(CT=(C_1,C_2),d_I)\rightarrow m$

Thoughts

基于CBDH问题: 给定问题实例$(g,g^a,g^b,g^c)$, 计算$e(g,g)^{abc}$.

  • 密文构造
    \begin{eqnarray}
    H_2(e(g,g)^{abc})\oplus m&=&H_2(e(g^a,g^b)^c)\oplus m \nonumber \\
    &=&H_2(e(g_1,H_1(I^\ast))^s)\oplus m. \nonumber
    \end{eqnarray}
  • 问题镶嵌
    • 设置$g_1=g^a$, 暗含$\alpha=a$但是不被$\mathcal{B}$知道.
    • 设置$H_1(I^\ast)=g^b$, $I^\ast$对应的私钥$d_{I^\ast}=g^{ab}$ (实际上被嵌入了$\mathbb{G}$群中的CDH问题). 为使$\mathcal{B}$能够回答私钥查询, 设置$H_1(I_i)=g^{x_i},x_i\xleftarrow{$}\mathbb{Z}_p,I_i\neq I^\ast$. 因此, 猜测$\mathcal{A}$向$H_1(\cdot)$进行的第$i^\ast$个查询的输入作为挑战身份, 设置随机预示机$H_1(\cdot)$为
      \begin{equation}
      H_1(I_i)=\left\{
      \begin{aligned}
      &g^{x_i},i\neq i^\ast \\
      &g^b,i=i^\ast
      \end{aligned}
      \right. \nonumber
      .
      \end{equation}

Proof

Given a problem instance $(g,g^a,g^b,g^c)$, we construct a simulator $\mathcal{B}$ to compute $e(g,g)^{abc}$.

Setup: $\mathcal{B}$ sets $g_1=g^a$, and sends $g_1$ to $\mathcal{A}$.

H-Query:

  • $H_1(\cdot)$. $\mathcal{B}$ guesses an index $i^\ast\in\{1,2,\cdots,q_{H_1}\}$, chooses $x_i\xleftarrow{$}\mathbb{Z}_p,i\in\{1,2,\cdots,q_{H_1}\}$, and then sets $H_1(\cdot)$ as
    \begin{equation}
    H_1(I_i)=\left\{
    \begin{aligned}
    &g^{x_i},i\neq i^\ast \\
    &g^b,i=i^\ast
    \end{aligned}
    \right. \nonumber
    .
    \end{equation}
  • $H_2(\cdot)$. $H_2(y_j)=Y_j,j\in\{1,2,\cdots,q_{H_2}\},Y_j\xleftarrow{$}\{0,1\}^n$.

Phase1:

  • Private Key Query. For a private key query on identity $I$, if $I=I_{i^\ast}$, $\mathcal{B}$ aborts. Otherwise, suppose that $I=I_i$ for some $i$, $\mathcal{B}$ sets $d_I=d_{I_i}=g_1^{x_i}=(g^{x_i})^a=H_1(I_i)^\alpha$.

Challenge: $\mathcal{A}$ outputs the challenge identity $I^\ast$ along with two distinct messages $m_0,m_1$. If $I^\ast\neq I_{i^\ast}$, $\mathcal{B}$ aborts. Otherwise, $\mathcal{B}$ chooses $coin\xleftarrow{$}\{0,1\},R\xleftarrow{$}\{0,1\}^n$, sets the challenge ciphertext as $CT^\ast=(g^c,R)$ and sents $CT^\ast$ to $\mathcal{A}$.

Phase2: The same as Phase1 with the restriction that no private key query is allowed on $I^\ast$.

Guess: $\mathcal{B}$ randomly chooses one hash query input to $H_2(\cdot)$ as the challenge hash query. The challenge hash query: $e(g_1,H_1(I^\ast))^c=e(g,g)^{abc}$. Loss: $q_{H_1}\cdot q_{H_2}$.
