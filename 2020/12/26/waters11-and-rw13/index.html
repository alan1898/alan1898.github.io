Waters11 and RW13

Posted on 2020-12-26

Two attribute-based encryption schemes.

Waters11 (CP)

Water11 is based on the partitioning strategy, the basis of partitioning for key generation query is as follows: suppose that there is an access structure $(M,\rho)$, for any unauthorized set of rows $I$, there exists a vector $\vec{w}$, such that $w_1\neq0$ and $\vec{w}\cdot \vec{M_i}=0$ for any $i\in I$.

For a key generation query on an unauthorized set $S$ of attributes, the simulator should create the corresponding private key. The main parts of private key we should pay attention to are $K=g^\alpha g^{at}$ and $K_x=h_x^t, \forall x\in S$.

  • Since $g^\alpha$ has a term of $g^{a^{q+1}}$, $t$ should have a term of $-a^q$ in order for $K$ being simulated successfully. We let $w_1=-1$ to embed the partitioning strategy, and then $t$ contains a term of $w_1a^q$.

  • For each attribute $x\in S$, let $X$ be the set $\{i:\rho^\ast(i)=x,i\in[l^\ast]\}$ and we should simulate $h_x^t$.

    • If $X=\varnothing$, $x$ does not influence the partitioning and $h_x$ can be any random value.
    • If $X\neq\varnothing$, $x$ is related to some rows of $M^\ast$. We let $h_x^t$ contain something like $g^{\vec{w}\cdot\vec{M_i}\cdot a^{q+1}}$ to achieve the partitioning strategy. So $w_1a^q$ should be related with $aM^\ast_{i,1}$, and then we can assign The simulator setsand(which is also suitable for the first case). The existance of the ‘$/b_i$’ part is due to the simulation of $C_i$ during the challenge phase, and the additional items caused by this should be added to the assumption. When $\rho(\cdot)$ is required to be an injective function, the simulator can setwhere $\rho^\ast(i)=x$, and we can use weaker assumption (the construction can also be simplified).
