An IND-sID-CPA-Secure IBE scheme

The main challenge to construct an IND-sID-CPA-Secure IBE scheme is to ensure that any PPT adversary cannot produce a secret key for a new identity even if it is given the access to the secret key generation oracle KeyGen$(\cdot)$ compared with a CPA-Secure PKE scheme. In the security reduction, the new secret key should be used to solve a computational hard problem.

Using the DBDH problem

For an instance of the DBDH problem $(g,g^a,g^b,g^c,T)$, the task is to determine whether $T$ is equal to $e(g,g)^{abc}$ or is selected uniformly from $\mathbb{G}_T$. Under the assumption that the DBDH problem is hard, the CDH problem (given $(g,g^a,g^b)$, compute $Z=g^{ab}$) in group $\mathbb{G}$ must also be hard. Otherwise, the DBDH problem can be solved effectively by computing $e(g^{ab},g^c)=g^{abc}$ and comparing the result with $T$.

We can set the secret key for identity $I\in\mathbb{Z}_p^\ast$ as $g_2^\alpha\cdot(u^Iv)^r$, where $g_2,u,v$ are from the public parameters, and $r$ is choosen uniformly from $\mathbb{Z}_p^*$. In the security reduction, $\mathcal{B}$ sets $g_2=g^b$, and implicitly sets $\alpha=a$ (but unknown to $\mathcal{B}$). Thus, $g_2^\alpha=g^{ab}$, which is the solution to the CDH problem.

In order to answer the secret key queries made by $\mathcal{A}$, $\mathcal{B}$ should adjust $(u^Iv)^r$ so that $g_2^\alpha\cdot(u^Iv)^r$ does not have the power of $ab$ (base $g$). Suppose that $u=g^x,v=g^y$ for some $x,y$, then $g_2^\alpha\cdot (u^Iv)^r=g^{ab+(xI+y)r}$.

Suppose that $\mathcal{A}$ launches a successful attack on a challenge identity $I^\ast$, that is, it outputs $g_2^\alpha\cdot(u^{I^\ast}v)^{r^\ast}=g^{ab}\cdot g^{(xI^\ast+y)r^\ast}$. If $\mathcal{B}$ can compute $g^{(xI^\ast+y)r^\ast}$ by itself, the attack can be reduced to solving the CDH problem by computing $\frac{g^{ab}\cdot g^{(xI^\ast+y)r^\ast}}{g^{(xI^\ast+y)r^\ast}}$. If $g^{xI^\ast+y}$ does not have the power of $a$ or $b$ and we set $g^r$ as a part of secret key in the real scheme, $\mathcal{B}$ can compute $g^{(xI^\ast+y)r^\ast}$ by itself.

如果$x$和$y$均不含$a$, 也不含$b$, 那么为了使私钥$g^{ab+(xI+y)r}$不含$ab$的幂次, $r$需要包含一个$ab$的某个负数倍数, 因此$\mathcal{B}$无法计算$g^r$.

如果$y$含$a$或$b$, 那么$g^{xI^\ast+y}$含$a$或$b$的幂次, 攻击将无法被归约到解决困难问题上.

特别地, 当$y=0$且$x$含$a$或$b$时, 此时私钥结构为$g_2^\alpha\cdot(u^I)^r$, 那么$g^{xI^\ast}$含$a$或$b$的幂次, 攻击将无法被归约到解决困难问题上.

选择随机元素$t_1,t_2\xleftarrow{$}\mathbb{Z}_p^\ast$, 令$x=t_1a,y=-xI^\ast+t_2=-t_1aI^\ast+t_2$. 可以看出, 该归约过程需要敌手首先声明所要攻击的身份.

  • 对于私钥查询: $g^{ab}\cdot g^{(xI+y)r}=g^{ab+(t_1a(I-I^\ast)+t_2)r}$. 选择随机元素$\tilde{r}\xleftarrow{$}\mathbb{Z}_p^\ast$, 令$r=-\frac{b}{t_1(I-I^\ast)}+\tilde{r}$.
    \begin{eqnarray}
    g^{ab}\cdot g^{(xI+y)r}&=&g^{ab+(t_1a(I-I^\ast)+t_2)r} \nonumber \\
    &=&g^{ab+(t_1a(I-I^\ast)+t_2)(-\frac{b}{t_1(I-I^\ast)}+\tilde{r})} \nonumber \\
    &=&g^{t_1a(I-I^\ast)\tilde{r}-\frac{bt_2}{t_1(I-I^\ast)}+t_2\tilde{r}} \nonumber \\
    &=&(g^a)^{t_1(I-I^\ast)\tilde{r}}(g^b)^{-\frac{t_2}{t_1(I-I^\ast)}}g^{t_2\tilde{r}}. \nonumber
    \end{eqnarray}
    可以被$\mathcal{B}$计算出来. 同时,
    \begin{eqnarray}
    g^r&=&g^{-\frac{b}{t_1(I-I^\ast)}+\tilde{r}} \nonumber \\
    &=&(g^b)^{-\frac{1}{t_1(I-I^\ast)}}g^\tilde{r}. \nonumber
    \end{eqnarray}
    可以被$\mathcal{B}$计算出来.

  • 对于攻击:
    \begin{eqnarray}
    g^{xI^\ast+y}&=&g^{t_1aI^\ast-t_1aI^\ast+t_2} \nonumber \\
    &=&g^{t_2}. \nonumber
    \end{eqnarray}
    可以被$\mathcal{B}$计算出来. 因此,
    \begin{eqnarray}
    \frac{g^{ab}\cdot g^{(xI^\ast+y)r^\ast}}{(g^{r^\ast})^{t_2}}&=&\frac{g^{ab}\cdot g^{t_2r^\ast}}{g^{t_2r^\ast}} \nonumber \\
    &=&g^{ab}. \nonumber
    \end{eqnarray}
