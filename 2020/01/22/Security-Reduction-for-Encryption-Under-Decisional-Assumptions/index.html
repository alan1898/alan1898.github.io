Security Reduction for Encryption Under Decisional Assumptions

Posted on 2020-01-22

Suppose there exists an adversary $\cal A$ who can break the proposed encryption scheme in the corresponding security model. We construct a simulator $\cal B$ to solve a decisional hard problem. $\cal B$ does not know how $\cal A$ breaks the proposed encryption scheme but only receives the guess $c’$ of $c$ from $\cal A$.

How the simulator use the adversary’s attack to break the underlying decisional hard problem

The method of guessing $Z$ is the same in all security reductions. To be precise, if $c’=c$， the simulator outputs that $Z$ is true. Otherwise, $c’\neq c$, and it outputs that $Z$ is false.

Simulation of the challenge ciphertext

The simulator must embed the target $Z$ in the challenge ciphertext such that it satisfies the following conditions.

  • If $Z$ is true, the true challenge ciphertext is a correct ciphertext whose encrypted message is $m_c\in\{m_0,m_1\}$. The simulation should be indistinguishable and the adversary should be able to guess the encrypted message correctly with non-negligible advantage defined in the breaking assumption.
  • If $Z$ is false, the false challenge ciphertext (can be either correct or incorrect) cannot be an encryption of the message $m_c$ from the point of view of the adversary. The adversary should not be able to guess the encrypted message correctly except with negligible advantage.
We program the simulation in such a way for the reason that we hope to maximize the advantages of the simulator solving the underlying hard problem.

The advantage of solving the underlying decisional hard problem

$Solve$: The simulator solves the underlying decisional hard problem.
$Guess_{True}$: The simulator guesses $Z$ is true.
$Guess_{False}$: The simulator guesses $Z$ is false.
The advantage of solving the underlying hard problem is
\begin{eqnarray}
\varepsilon_R &=& Pr[Solve|Z=True]-\frac12+Pr[Solve|Z=False]-\frac12 \nonumber \\
&=& Pr[Guess_{True}|Z=True]-\frac12+Pr[Guess_{False}|Z=False]-\frac12 \nonumber \\
&=& Pr[Guess_{True}|Z=True]-(1-Pr[Guess_{False}|Z=False]) \nonumber \\
&=& Pr[Guess_{True}|Z=True]-Pr[Guess_{True}|Z=False]. \nonumber
\end{eqnarray}
$SS$: The event of successful simulation.
$US$: The event of unsuccessful simulation.
According to the way that the simulator use the adversary’s attack to break the underlying decisional hard problem, we have
\begin{eqnarray}
Pr[Guess_{True}|SS]=Pr[c’=c]. \nonumber
\end{eqnarray}
If the simulation is unsuccessful, the simulator will randomly guess $Z$ by itself, and thus we have
\begin{eqnarray}
Pr[Guess_{True}|US]=\frac12. \nonumber
\end{eqnarray}
$P_T$: The probability of the adversary breaking the challenge ciphertext if $Z$ is true.
$P_F$: The probability of the adversary breaking the challenge ciphertext if $Z$ is false.
By applying the law of total probability, we have
\begin{eqnarray}
&&Pr[Guess_{True}|Z=True] \nonumber \\
&=&Pr[Guess_{Ture}|Z=True\land SS]Pr[SS]+Pr[Guess_{Ture}|Z=True\land US]Pr[US] \nonumber \\
&=&Pr[c’=c|Z=True]Pr[SS]+\frac12Pr[US] \nonumber \\
&=&P_T\cdot P_S+\frac12(1-P_S). \nonumber
\end{eqnarray}
\begin{eqnarray}
&&Pr[Guess_{True}|Z=False] \nonumber \\
&=&Pr[Guess_{Ture}|Z=False\land SS]Pr[SS]+Pr[Guess_{Ture}|Z=False\land US]Pr[US] \nonumber \\
&=&Pr[c’=c|Z=False]Pr[SS]+\frac12Pr[US] \nonumber \\
&=&P_F\cdot P_S+\frac12(1-P_S). \nonumber
\end{eqnarray}
So the advantage of solving the underlying hard problem is
\begin{eqnarray}
\varepsilon_R = P_S(P_T-P_F). \nonumber
\end{eqnarray}
